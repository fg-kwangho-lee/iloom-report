결제 병합 기능 악용
— 횡령 위험
로드매장
기능 설명
결제 병합은 한 고객의 여러 수주를 하나의 결제로 묶는 기능.
정상 사용 케이스 3가지:
① 소파수수 자동분리
목재품목+소파 동시 구매 시 소파전담 시공팀 운영으로 수주 자동분리 → 한 결제내역으로 병합 필요
② 프로모션으로 인한 수주 분리
각기 다른 프로모션 수주를 분리 등록한 경우 → 하나의 결제내역으로 병합 필요
③ 당일 반품 후 재수주
시공 당일 반품 시 양지센터 입고까지 최대 7일 소요 → 납기 확보를 위해 신규 재수주 등록 후, 원수주 반품매출 완료 시점에 원수주와 결제 병합 진행
발생 영향
직원 횡령 가능 채권 정산 혼선 병합 건수 무제한

문제의 근본 원인
결제 병합 시 고객 동일성 검증 로직이 없어 이미 결제가 완료된 수주를 타인의 수주에 자유롭게 병합 가능.
병합 건수 상한도 없어 금액 누락 추적이 어려워 채권 정산 불일치로 이어짐.
2
입점몰 결제 수기입력
— 불일치·횡령 위험
입점몰 (백화점·아울렛)
발생 영향
상품권 횡령 사례 발생 POS-C+ 금액 불일치 EXTRA 수주 악용
IT 요청 사항
근본 해결: 결제영수증 스캔 시 C+ 자동 입력 개발
현실적 대안: 월 단위 수기입력 vs POS 대조 화면 신설
현황
입점몰 POS와 C+가 미연동되어, 결제 후 직원이 영수증 보고 수기 입력하는 구조. 검증 없는 수기 입력이 아래 3가지 문제로 이어짐.
발생한 문제 3가지
① POS-C+ 불일치 — 수기 오입력으로 매출금액 차이 발생
② 상품권 횡령 (신세계파주) — 고객 상품권 수령 후 일마감 미제출, 수기로만 입력
③ EXTRA 수주 악용 — 초과 상태 수주에 전시품·판촉물 병합 후 미결제

③ 납기분할·프로모션 수주에서 끝전 차이 발생
