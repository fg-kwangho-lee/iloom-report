매장 유형별 오류 상세

로드매장

독립 POS 사용 · 커넥트플러스 직접 연동 매장

1 결제 병합 건수 제한 부재
병합 건수 무제한으로 10건 이상 병합 시 미결·초과 확인이 어려워 관리 공백 발생
횡령 악용 — 납기 임박 미결 수주에 완결 건을 병합 후 현금 가로채는 사례 (천안불당)
→ 대리점 결제 병합 횟수 상한 제한 (IT 의뢰 진행 중)
→ 근본 원인: 반품매출이 당일 처리되지 않아 반품 후 재수주가 반복되는 구조에서 비롯 — 당일 반품매출 발행 시스템 개선이 병합 이슈의 근본 해결책
2 e-iloom → 커넥트플러스 전환기 매칭 불가
2020년 12월 이전 수주 건 상세내역 조회 불가
대표자·코드 변경 매장의 전시품 판매 내역 소급 확인 불가
→ 프로그램 종료로 개선 불가 — 해당 기간 건들 별도 수기 정리 필요
3 부가세 계산 오류 (원단위 단수 처리)
반품·AS환불·프로모션·납기분할 시 ERP와 커넥트플러스의 원단위 반올림 기준이 달라 금액 불일치 발생
일반 수주에서도 동일 문제로 외상매출금 항목 간 차액 발생
→ 커넥트플러스–ERP 부가세 반올림 기준 통일 (시스템 개선)
4 미결 / 초과 상태 오류
추가시공 마이너스 수주 누락 — 당일 취소 건의 반품 수주가 수기 누락
강제 결제취소 미연동 — 관리자 페이지 강제취소가 커넥트플러스에 미반영
단가정정 오류 — 정정 후에도 미결 상태 잔류
→ 추가시공 반품 자동화 / 강제취소 자동 매칭 시스템 개선
5 환불 처리 누락
반품매출 지연 — 입고 전 매장 추가결제 후 매장결제분 환불 미진행
전시품·판촉물 매장 결제분 환불 단순 누락
박람회 계약금 등 기타 환불 미진행
→ 당일 반품 당일 발행 / 환불 누락 알림 시스템 도입
6 AS 환불 수주 등록 오류
대리점 오등록 — 컨택센터 발행 시 다른 대리점 코드에 AS환불 수주가 잘못 등록
부가세 누락 — 등록 시 부가세 없이 수주 생성
→ AS환불 수주 등록 시 원수주 자동 연결 시스템 구현
7 거래처 원장 차·대변 역전 오류
ERP ACC 오류로 반품수수료 등 대변 항목이 차변에 기입되는 현상 발생
포항·대전도안·천안불당 등 정산 시 발생 확인
→ ERP ACC 전표 생성 로직 수정 (시스템 개선)
입점몰

백화점·아울렛 등 입점 형태 · 자체 POS 사용 불가

1 결제내역 수기입력으로 인한 불안정성
입점몰 POS와 커넥트플러스 미연동 → 영수증 기반 수기입력으로 신뢰도 저하
횡령 사례 — 신세계파주 직원이 고객 상품권 수령 후 일마감 미진행 횡령 (신세계파주)
→ 영수증·결제수단 스캔 시 커넥트플러스 자동 연동 시스템 개발 (근본 대안)
2 전시품 판매 정산 이슈
중복 정산 — 전시품 판매 취소 내역이 본사에 미공유될 경우 동일 건 이중 정산 발생
EXTRA 수주 악용 — 서브계정 초과 상태 EXTRA 수주에 전시품·판촉물 수주 병합 후 미결제 (롯데청량리 사례)
→ 입점몰 전시품 판매 시 일룸 POS 별도 사용 / EXTRA 수주 월별 점검
3 입점몰 법인 일괄 정산으로 인한 채권 오류
신세계 백화점 본사가 일룸 법인 산하 여러 브랜드 매출을 통합 단일 코드로 일괄 입금
재경팀의 거래처별 대체전표 누락 시 채권 불일치 발생 — 기말 조정 방식으로 운영 중
신세계 하남(팝업) 사례 — 기말 조정 시 해당 코드 누락
→ 신세계 측에 브랜드별 별도 정산 요청 / 재입점 시 대체전표 즉시 발행 프로세스 수립
4 파트너포탈 건별 매출 조회 불가
입점몰 파트너포탈은 일별 합계만 제공 — 건별 상세 조회 불가
정산 시 커넥트플러스 입력금액과 POS 금액을 건별 직접 비교 불가
→ 월별 커넥트플러스 입력금액 vs POS 총액 대조 점검 (현실적 대안)
